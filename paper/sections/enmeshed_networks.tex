\section{Enmeshed Networks}
\label{sec:enmeshed}

An enmeshed network is a lightweight parametric module that shares the forward pass of a frozen host model, reading intermediate representations at each layer and writing modulations back into the same computation. The enmeshed module maintains its own parameters, receives its own input (the \emph{parallel context}), and can be cleanly removed to recover the unmodified host. The key distinction from existing adaptation methods is \emph{where} and \emph{how} the module interfaces with the host:

\begin{itemize}[leftmargin=*]
    \item \textbf{Context-level methods} (RAG, Reflexion, MemGPT) add tokens to the host's input. Memory competes for attention with everything else and dilutes with length.
    \item \textbf{Parameter-level methods} (LoRA, adapters) modify the host's weights. Each adaptation requires gradient descent. The result is fixed after training.
    \item \textbf{Activation-level methods} (RepEng, ActAdd) inject fixed directional vectors. The same vector is applied regardless of experience.
    \item \textbf{Enmeshed networks} process a parallel input through the host's own layers, using the resulting hidden states as a bridge to external knowledge. Two mechanisms emerge: \emph{(a)}~compile hidden states into per-layer modulation tensors that shape the primary context's attention (dispositional tasks); \emph{(b)}~project hidden states into a retrieval space to fetch relevant knowledge nodes from a persistent graph (agentic tasks). Both are experience-specific, inference-cost adaptable, and operate outside the context window.
\end{itemize}

\subsection{Design Space}

We characterise enmeshed networks along six axes (Appendix Table~\ref{tab:design_space}): \textbf{blending function} (Level 1 additive in HEXIS, Levels 2-3 are gated/cross-attention), \textbf{rank} ($r=16$; rank scales as $r/d \approx 0.006$), \textbf{modulation targets} (Q+V), \textbf{patching pattern} (stride-3, 11 of 32 Qwen3.5-4B-Base layers), \textbf{compilation} (conviction-weighted), and \textbf{temporal profile} (constant with phase-gated mitigation). At rank 16, modulation adds 81{,}920 FLOPs per layer per token --- approximately 200$\times$ cheaper than adding equivalent belief tokens to context.

\subsection{The Mind Tree}
\label{sec:mind_tree}

The enmeshed network requires a structured input for its parallel context channel. The \emph{Mind Tree} is a typed memory schema with hierarchical nodes organised into six sections (\texttt{identity}, \texttt{beliefs}, \texttt{strategies}, \texttt{memories}, \texttt{models}, \texttt{values}; full role table in Appendix~\ref{sec:mind_tree_spec}). Each belief node carries structured metadata: categorical conviction (\texttt{strong}, \texttt{moderate}, \texttt{agnostic}), domain tags, an \texttt{addresses} field listing query types the node answers, and a \texttt{novel} flag marking content unlikely to survive compilation. This metadata enables both \emph{compilation weighting} (high-conviction nodes receive higher weight during $\phi$'s pooling step) and \emph{deterministic curation} (novel content is routed to the explicit slot rather than the compiled channel).

\textbf{Categorical vs.\ numeric conviction.} We use categorical conviction labels rather than numeric credences. This is an architectural decision confirmed by a negative result: numeric credence values are nearly indistinguishable in the transformer's attention space (Appendix~\ref{sec:credence_negative}).
